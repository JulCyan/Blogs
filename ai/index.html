# AI 学习笔记

这一组文档不是单纯的复现教程，也不是零散随笔。

它记录的是一条真实的学习路径：

  • 怎么把 AI Agent 系统先跑起来
  • 怎么判断哪里是真的问题，哪里只是正常现象
  • 怎么在踩坑之后形成更稳的推荐路径

所以这套内容会同时保留三层结构：

  • 主流程：最短可复现路径
  • 问题锚点 / QA：遇到分支问题时直接跳转
  • 踩坑建议：记录为什么最后推荐这样做

# 阅读顺序

如果你是第一次看，建议按这个顺序读：

  1. openclaw.md
  2. models.md
  3. telegram.md
  4. 后续再补 Workspace、踩坑整理

这样读的好处是：

  • 先理解 OpenClaw 是什么、怎么判断它跑起来了
  • 再理解模型层为什么会出问题
  • 再进入具体渠道接入
  • 最后再补目录结构、长期协作方式

# 当前重点

# AI Agent 元工作区 (opens new window)

这一块虽然还没有单独拆成文档，但它应该被视为 AI 专题里的重点内容，而不只是一个旁支补充。

它更接近你对 AI 的深层理解来源：

  • 不只是“怎么调用模型”
  • 而是“怎么组织 Agent、规则、上下文、技能、工作流和长期协作”
  • 也更接近真实 AI Agent系统背后的规范、约定与协议

所以当前 AI 区的理解顺序应该是：

  • OpenClaw 负责建立运行环境和系统入口
  • Models 负责理解模型层与调用链路
  • Telegram 负责理解渠道接入
  • AI Agent 元工作区 则代表更深入的工程化组织方式

后续这部分值得单独展开成专题文档。
